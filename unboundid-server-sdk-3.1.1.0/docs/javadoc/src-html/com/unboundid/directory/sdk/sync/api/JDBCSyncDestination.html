054    /**
055     * This class defines an API that must be implemented by extensions
056     * in order to synchronize data into a relational database. Since the UnboundID
057     * Synchronization Server is LDAP-centric,
058     * this API allows you to take LDAP entries and split them out into
059     * database content and make the appropriate updates. The lifecycle of a sync
060     * operation is as follows:
061     * <ol>
062     * <li>Detect change at the synchronization source</li>
063     * <li>Fetch full source entry</li>
064     * <li>Perform any mappings and compute the equivalent destination entry</li>
065     * <li>Fetch full destination entry</li>
066     * <li>Diff the computed destination entry and actual destination entry</li>
067     * <li>Apply the minimal set of changes at the destination to bring it in sync
068     * </li>
069     * </ol>
070     * This implies that the {@link #fetchEntry(TransactionContext, Entry,
071     * SyncOperation)} method will
072     * always be called once prior to any of the other methods in this class.
073     * <p>
074     * In several places a {@link TransactionContext} is provided, which allows
075     * controlled access to the target database. By default, methods in this class
076     * are always provided with a fresh connection (i.e. a new transaction), and the
077     * Synchronization Server will always commit or rollback the transaction
078     * automatically, depending on whether the method returned normally or threw an
079     * exception. Implementers may optionally perform their own transaction
080     * management within these methods if necessary. The TransactionContext also
081     * provides logging facilities and allows a timeout, which will automatically
082     * invalidate the context object after the specified duration.
083     * <p>
084     * Several of these methods throw {@link SQLException}, which should be used in
085     * the case of any database access error. For other types of errors, runtime
086     * exceptions may be used (IllegalStateException, NullPointerException, etc.).
087     * The Synchronization Server will automatically retry operations that fail, up
088     * to a configurable amount of attempts. The exception to this rule is if a
089     * SQLException is thrown with a SQL state string beginning with "08"; this
090     * indicates a connection error, and in this case the operation is retried
091     * indefinitely.
092     * <p>
093     * Implementers need not worry about the cost of repeatedly parsing and
094     * preparing SQL statements; all statements will be automatically cached.
095     * <BR>
096     * <H2>Configuring JDBC Sync Destinations</H2>
097     * In order to configure a JDBC sync destination based on this API and
098     * written in Java, use a command like:
099     * <PRE>
100     *      dsconfig create-sync-destination \
101     *           --destination-name "<I>{destination-name}</I>" \
102     *           --type third-party-jdbc \
103     *           --set "server:{server-name}" \
104     *           --set "extension-class:<I>{class-name}</I>" \
105     *           --set "extension-argument:<I>{name=value}</I>"
106     * </PRE>
107     * where "<I>{destination-name}</I>" is the name to use for the JDBC sync
108     * destination instance, "<I>{server-name}</I>" is the name of the JDBC external
109     * server that will be used as the sync destination, "<I>{class-name}</I>" is
110     * the fully-qualified name of the Java class written using this API, and
111     * "<I>{name=value}</I>" represents name-value pairs for any arguments to
112     * provide to the JDBC sync destination.  If multiple arguments should be
113     * provided to the JDBC sync destination, then the
114     * "<CODE>--set extension-argument:<I>{name=value}</I></CODE>" option should be
115     * provided multiple times.
116     */
117    @Extensible()
118    @SynchronizationServerExtension(appliesToLocalContent=false,
119         appliesToSynchronizedContent=true)

173       * This hook is called when a Sync Pipe first starts up, or when the
174       * <i>resync</i> process first starts up. Any initialization of this sync
175       * destination should be performed here. This method should generally store

177       * member so that it can be used elsewhere in the implementation.
178       * <p>
179       * The default implementation is empty.
180       *
181       * @param ctx
182       *          a TransactionContext which provides a valid JDBC connection to the

184       * @param  serverContext  A handle to the server context for the server in
185       *                        which this extension is running.
186       * @param  config         The general configuration for this sync destination.
187       * @param  parser         The argument parser which has been initialized from
188       *                        the configuration for this JDBC sync destination.
189       */

191      public void initializeJDBCSyncDestination(
192                                             final TransactionContext ctx,
193                                             final SyncServerContext serverContext,
194                                             final JDBCSyncDestinationConfig config,
195                                             final ArgumentParser parser)
196      {
197        // No initialization will be performed by default.
198      }

222      /**
223       * Return a full destination entry (in LDAP form) from the database,
224       * corresponding to the the source {@link Entry} that is passed in.
225       * This method should perform any queries necessary to gather the latest
226       * values for all the attributes to be synchronized and return them in an
227       * Entry.
228       * <p>
229       * Note that the if the source entry was renamed (see
230       * {@link SyncOperation#isModifyDN}), the
231       * <code>destEntryMappedFromSrc</code> will have the new DN; the old DN can
232       * be obtained by calling
233       * {@link SyncOperation#getDestinationEntryBeforeChange()} and getting the DN
234       * from there. This method should return the entry in its existing form
235       * (i.e. with the old DN, before it is changed).
236       * <p>
237       * This method <b>must be thread safe</b>, as it will be called repeatedly and
238       * concurrently by each of the Sync Pipe worker threads as they process
239       * entries.
240       * @param ctx
241       *          a TransactionContext which provides a valid JDBC connection to the
242       *          database.
243       * @param destEntryMappedFromSrc
244       *          the LDAP entry which corresponds to the database "entry" to fetch
245       * @param  operation
246       *          the sync operation for this change
247       * @return a full LDAP Entry, or null if no such entry exists.
248       * @throws SQLException
249       *           if there is an error fetching the entry
250       */
251      @ThreadSafety(level = ThreadSafetyLevel.METHOD_THREADSAFE)
252      public abstract Entry fetchEntry(final TransactionContext ctx,
253                                       final Entry destEntryMappedFromSrc,
254                                       final SyncOperation operation)
255                                                  throws SQLException;
256

259      /**
260       * Creates a full database "entry", corresponding to the the LDAP
261       * {@link Entry} that is passed in. This method should perform any inserts and
262       * updates necessary to make sure the entry is fully created on the database.
263       * <p>
264       * This method <b>must be thread safe</b>, as it will be called repeatedly and
265       * concurrently by the Sync Pipe worker threads as they process CREATE
266       * operations.
267       * @param ctx
268       *          a TransactionContext which provides a valid JDBC connection to the
269       *          database.
270       * @param entryToCreate
271       *          the LDAP entry which corresponds to the database "entry" to create
272       * @param  operation
273       *          the sync operation for this change
274       * @throws SQLException
275       *           if there is an error creating the entry
276       */
277      @ThreadSafety(level = ThreadSafetyLevel.METHOD_THREADSAFE)
278      public abstract void createEntry(final TransactionContext ctx,
279                                       final Entry entryToCreate,
280                                       final SyncOperation operation)
281                                          throws SQLException;
282    
283    
284    
285      /**
286       * Modify an "entry" in the database, corresponding to the the LDAP
287       * {@link Entry} that is passed in. This method may perform multiple updates
288       * (including inserting or deleting rows) in order to fully synchronize the
289       * entire entry on the database.
290       * <p>
291       * Note that the if the source entry was renamed (see
292       * {@link SyncOperation#isModifyDN}), the <code>fetchedDestEntry</code> will
293       * have the old DN; the new DN can be obtained by calling
294       * {@link SyncOperation#getDestinationEntryAfterChange()} and getting the DN
295       * from there.
296       * <p>
297       * This method <b>must be thread safe</b>, as it will be called repeatedly and
298       * concurrently by the Sync Pipe worker threads as they process MODIFY
299       * operations.
300       * @param ctx
301       *          a TransactionContext which provides a valid JDBC connection to the
302       *          database.
303       * @param fetchedDestEntry
304       *          the LDAP entry which corresponds to the database "entry" to modify
305       * @param modsToApply
306       *          a list of Modification objects which should be applied
307       * @param  operation
308       *          the sync operation for this change
309       * @throws SQLException
310       *           if there is an error modifying the entry
311       */
312      @ThreadSafety(level = ThreadSafetyLevel.METHOD_THREADSAFE)
313      public abstract void modifyEntry(final TransactionContext ctx,
314                                       final Entry fetchedDestEntry,
315                                       final List<Modification> modsToApply,
316                                       final SyncOperation operation)
317                                              throws SQLException;
318    
319    
320    
321      /**
322       * Delete a full "entry" from the database, corresponding to the the LDAP
323       * {@link Entry} that is passed in. This method may perform multiple deletes
324       * or updates if necessary to fully delete the entry from the database.
325       * <p>
326       * This method <b>must be thread safe</b>, as it will be called repeatedly and
327       * concurrently by the Sync Pipe worker threads as they process DELETE
328       * operations.
329       * @param ctx
330       *          a TransactionContext which provides a valid JDBC connection to the
331       *          database.
332       * @param fetchedDestEntry
333       *          the LDAP entry which corresponds to the database "entry" to delete
334       * @param  operation
335       *          the sync operation for this change
336       * @throws SQLException
337       *           if there is an error deleting the entry
338       */
339      @ThreadSafety(level = ThreadSafetyLevel.METHOD_THREADSAFE)
340      public abstract void deleteEntry(final TransactionContext ctx,
341                                       final Entry fetchedDestEntry,
342                                       final SyncOperation operation)
343                                            throws SQLException;
